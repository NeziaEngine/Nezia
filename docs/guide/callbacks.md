# イベントとコールバック

サウンドスレッドで起こったこと（再生終了など）をメインスレッドで受け取る仕組み。

## 毎フレーム `poll_events()` を呼ぶ

```rust
loop {
    // ... ゲームの更新 ...
    engine.poll_events();
}
```

- 溜まったイベントを取り出し、登録済みのコールバックを呼び出す。
- 呼ばないとコールバックは永久に呼ばれず、クロージャが解放されない。
- コールバックは `poll_events()` を呼んだスレッドで実行される。

公式 Integration（Unity / Unreal / Bevy 等のバインディング）ではこの呼び出しを
ホストの更新ループにフックして吸収しているため、利用者が自分で呼ぶ必要はない。
core を直接使う場合のみ自前のループに組み込むこと。

## コールバックの登録

`play_with_callback` / `play_to_bus_with_callback` でクロージャを渡す。

```rust
let on_finish = move || {
    println!("再生終わった");
};
engine.play_with_callback(buf, 1.0, 1.0, on_finish);
```

- クロージャは `FnOnce + Send + 'static`。
- 引数なし・戻り値なし。状態を伝えたいなら `Arc<AtomicBool>` などで共有する。

## 呼ばれるケース・呼ばれないケース

| ケース | 呼ばれる？ |
|---|---|
| 音が最後まで再生され、自然終了 | ✅ |
| `stop_all()` で打ち切り | ❌（解放はされる） |
| 同時発音数の上限で再生開始失敗 | ❌（解放はされる） |
| バスやバッファ無効で `play_*` が `false` | ❌（そもそも登録されない） |

つまり「終わったら必ず呼ぶ」ではなく「自然終了したら呼ぶ」コールバック。
内部の仕組みは [コールバック設計](../design/core/callbacks.md) を参照。
